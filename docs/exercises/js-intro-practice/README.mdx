### `helloName()`

- Define a function called `helloName()` that accepts 1 parameter ([arity](https://en.wikipedia.org/wiki/Arity) of 1).
- Use the symbol "name" for the parameter name.
- Return the string `"Hello, [name]!"` where `[name]` is the value passed to the function.

<details>

```js
function helloName(name) {

}
```

## Arrays

- [Arrays](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array) are ordered lists of values
- They share quite a few similarities to [Lists in Python](/docs/syllabus/#patterns)
- You can create an Array using the `[]` characters, with commas separating the individual values
- An Array can contain multiple types. ie: strings, numbers, boolean, etc...
- You can access individual values of an array using bracket notation, `array[0]`

### `threeFruits()`

## Objects

- Objects are collections of key/value pairs. They are similar (not the same!) to [Dictionaries in Python](/docs/syllabus/#patterns)
- You can create an Object using the `{}` brackets
- An Object can contain any type for each property. ie: strings, numbers, boolean, etc
- You can access individual values of an object using bracket notation, `myObject['key']`
- All Object keys are strings
- You can add properties to an Object by assigning a value to a new key

Useful reference:
[MDN: Objects](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object)

### Notation

:::tip
Q: How do I decide whether to use dot or bracket notation?

A: It depends, :grinnning:
Bracket notation is more flexible, but dot notation is a more convenient, readable syntax.

This MDN article is helpful: [MDN: Property Accessors](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Property_accessors)

:::

#### Bracket Notation

- Here is an object which contains a nested array
- Return the second item in the array `favoriteFoods`

```js
function nestedArray () {
    const bootcampInstructor = {
        name: 'Sean',
        favoriteColor: 'green',
        favoriteFoods: [
            'tacos',
            'pierogi',
            'waffles'
        ]
    }

    // Note that you can chain the bracket notation to reach into the Object AND into the Array.
    console.assert(bootcampInstructor['favoriteFoods'][0] === 'tacos')

    // Return the second item of the nested array favoriteFoods

}
```

<details>
<summary>Solution</summary>

```js
function nestedArray () {
    const bootcampInstructor = {
        name: 'Sean',
        favoriteColor: 'green',
        favoriteFoods: [
            'tacos',
            'pierogi',
            'waffles'
        ]
    }

    // Note that you can chain the bracket notation to reach into the Object AND into the Array.
    console.assert(bootcampInstructor['favoriteFoods'][0] === 'tacos')
```

#### Dot Notation

- Return the name of the `bootcampInstructor` Object below using dot notation

```js
function dotNotation () {
    const bootcampInstructor = {
        name: 'Sean',
        favoriteColor: 'green',
        favoriteFoods: [
            'tacos',
            'pierogi',
            'waffles'
        ]
    }

    // Some examples of using dot notation vs bracket notation:
    console.assert(bootcampInstructor.name === 'Susan')
    console.assert(bootcampInstructor['name'] === 'Susan')
    const nameString = 'name'
    console.assert(bootcampInstructor[nameString] === 'Susan')

    console.assert(bootcampInstructor.favoriteColor === 'green')
    console.assert(bootcampInstructor['favoriteColor'] === 'green')
    console.assert(bootcampInstructor['favorite' + 'Color'] === 'green')

    console.assert(bootcampInstructor['favoriteFoods'][0] === 'chicken pot pie')
    console.assert(bootcampInstructor.favoriteFoods[0] === 'chicken pot pie')

    // Return the name of the bootcampInstructor Object using dot notation

}
```

<details>
<summary>Solution</summary>
```js
function dotNotation () {
   const bootcampInstructor = {
        name: 'Sean',
        favoriteColor: 'green',
        favoriteFoods: [
            'tacos',

```
</details>
